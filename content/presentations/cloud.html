The cloud
First: why do we care?
the internet is just a bunch of computers
when you type in a URL, your browser sends a request to the computer located at the corresponding IP address
in response, that computer sends your computer HTML/CSS/JS to display
so, where are those computers?
they can be anywhere
your laptop can be a server (but not a very good one since it isn't always on)
you could leave a computer running in your basement
for higher-traffic sites, more computers are required to handle the load
that's essentially what people did: buy a bunch of computers and put them in a room
like this https://upload.wikimedia.org/wikipedia/commons/2/27/Wikimedia_Foundation_Servers-8055_08.jpg
https://upload.wikimedia.org/wikipedia/commons/1/10/Wikimedia_Foundation_Servers-8055_23.jpg
problems:
server administration is a specific skill
are your machines up-to-date?
server farms need significant cooling to operate
what if there's a spike in visitors?
what if the power goes out?
The cloud solution
let us worry about that for you
what do you get?
access to their data-center
datacenters that are optimized for cooling
ability to leverage additional computing power when traffic spikes
and not getting billed when you don't need that extra oomph
server configurations specialized for different functions: i.e. storage, compuation
promise of performance, and security
cloud products: X as a service
Infrastructure (IaaS): a bank of computers for you to configure manage
Platform (Paas): a configured cluster of computers for you to run your application/process/ product off of
Software (SaaS): a whole application that is accessed from a browser (i.e gmail)
developing on the cloud
AWS, Cloud Foundry, Heroku, Digital Ocean
typically, cost is usage-based
AWS
provides extremely wide range of specific services
Digital Ocean
provides Linux boxes for rent
Cloud Foundry & Heroku
deploy apps within a provided environment
Heroku deploys a git repository
CF deploys any specified file system
